Introduccion a los patrones de diseño de software

¿ Qué es la arquitectura de software?

Es la disciplina que se encarga de ver al software como un elemento integral, y desde ese punto de vista planificar un diseño de alto nivel, que será la estructura sobre la que se construirá el software.

Permite desarrollar productos más sofisticados y robustos, creando lineamientos claros permitiendo crear un código que permita obtener los mejoras resultados.

Al tener una visión completa, desde etapas tempranas ya se pueden incluir elementos claves, que implementarlos al final sería costoso.

Permite tomar a tiempo las decisiones críticas sobre la arquitectura de un programa que afectaran permanentemente la dirección de un proyecto.

El rol del arquitecto o arquitecta de software

No hay una definición exacta y normalizada.

Sin embargo hay características comunes, como por ejemplo tener una visión global del proyecto que les permita comprender tanto la capacidad técnica, las necesitadaes del negocio y los objetivos del producto, para poder crear estrategias coherentes que funcionen globalmente. También tienen que tener capacidad para crear estrategias y saber negociar para tenr una visión clara de como se debe llevar el proyecto; tomar decisiones críticas y servir de puente entre las necesidades de negocio y la factivilidad técnica.

Los arquitectos deben tener una excelente capacidad de análisis de problemas más complejos, para comprender claramente las consecuencias que tendrán sus decisiones..

No existe un camino perfecto, hay que lidiar con sus pros y sus contras.

El arquitecto debe tener un rol de liderazgo, debe sentar las bases para la visión del equipo de desarrollo, ser el ejemplo de las buenas prácticas.

¿Cómo funcionan los patrones de diseño de arquitectura?

Un patrón de diseño es una solución general para un problema que aparece ocasionalmente.

Los patrones de diseño nos ayudan a prevenir los errores antes de que ocurran y de paso pueden acelerar el proceso de desarrollo al proporcionar paradigmas de desarrollo probados muchas veces y con efectividad comprobada.

Utilizar patrones de diseño ayuda a evitar problemas sutiles (que pueden traer problemas mayores) y mejora la legibilidad del código para todos los desarrolladores que ya están familiarizados con estos patrones.

Los patrones de diseño se aplican al desarrollo de software y a la arquitectura general de un aplicación.
